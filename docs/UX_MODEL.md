# Automation Studio 全体UXモデル「シンプルかつ迷わない」

作成: 2026-07-05 / 設計: Claude Fable 5(Phase L-1)/ 実装前提: Codex 5.5

## 現状のUX問題(棚卸し)

1. **概念が多すぎる**: vol / チャンネル / workspace / intent / autopilot / ledger…。ユーザーが覚える語彙が多い。
2. **「次に何をすべきか」を画面が言わない**: 状態は見えるが、行動は自分で探す(在庫0の警告はあるが「作る」ボタンがその場に無い)。
3. **チャンネル文脈が二重**: ヘッダーの切替と、ページ内の「対象チャンネル」セレクタが併存。
4. **段階開示の不統一**: ページごとにタブ/折りたたみ/詳細ツールと流儀が違う。
5. **語彙の不統一**: 待機中/稼働中/dormant/ONチャンネル等、同じ概念に複数の言い方。

## 設計原則(5つ)

1. **1画面1つの問い** — 各画面はユーザーの1つの問いにだけ答える。
2. **Next Best Action** — 状態を示す全ての要素に「次にやること」ボタンを1つ添える。エラーも警告も、必ず行動に変換して見せる(error_humanizer思想の全面化)。
3. **チャンネル文脈は1箇所** — ヘッダーのチャンネル切替のみ。ページ内セレクタ廃止。「全チャンネル」横断ビューを既定に。
4. **カード規格の統一** — 全画面共通: タイトル+要約1行+CTA1つ。詳細はクリックで開く(段階開示は「折りたたみ」1種に統一)。
5. **語彙辞書** — UI全体で用語を統一: 自動運転 / 予約公開 / 在庫 / 検査 / 仕様書(TTP) / 需要。英語・内部語(dormant, ledger, workspace)はUIに出さない。
6. **絵文字を使わない** — UI全体で絵文字を使用しない。状態・レーン・操作の表現には SVG、CSS アイコン、または短いテキストラベルを使う。

## ナビゲーション(5→4)

| ナビ | 答える問い | 中身 |
|---|---|---|
| **ホーム** | 今どうなってる? 次に何する? | 状態サマリ+チャンネル行(自動運転トグル・在庫・収益化進捗を1行に統合)+Next Best Action列。**現在の「自動運転」ページを吸収** |
| **制作** | 動画はどこまで進んだ? 何を作る? | 動画一覧(工程進捗バー)+新規作成(vol / チャンネル)。現在の「動画」 |
| **分析** | 何が勝つ? | 現行のTTP中心構成(維持) |
| **設定** | 決めごとを変える | 現行の5タブ+ツール(維持) |

- ヘッダー: チャンネル切替(「全チャンネル」既定)+ 状態ピル(稼働中/停止中)+ 新規作成ボタンのみ。
- チャンネル行のNext Best Action例: 在庫0→「次のvolを作る」/ 要再認証→「再認証」/ 検査待ち→「検査を実行」/ 更新あり→「更新を適用」。

## 実装フェーズ(L-3)

1. L-3a: ホーム再構成(自動運転ページ吸収+チャンネル行+NBA)
2. L-3b: 制作(動画一覧の工程進捗バー化+旧詳細タブの整理)
3. L-3c: ヘッダー統一(チャンネル文脈一元化・ページ内セレクタ撤去)+語彙辞書適用
4. L-4: コード整理(監査TOP10、挙動変更なし)

各ステップで image2 モック→承認→実装→検証。デザイントークン(色・余白・カード規格)は共通CSSに一元定義。
