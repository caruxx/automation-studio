# AI BGM benchmark analysis concept

作成日: 2026-07-05
目的: Fable5、Gemini、Claude、Codex、ChatGPT など、どのAIモデルにも渡せる「AI BGMベンチマーク分析」の基本思想をまとめる。

## 基本思想

AI BGMのベンチマーク分析では、伸びているチャンネルを広く眺めるだけでは足りない。見るべき中心は「伸びた1本の動画」と「その動画を必要とした視聴者の場面」。

分析の目的は、ベンチマークをコピーすることではない。視聴者が反応した抽象パターンを分解し、自分のチャンネルに合う形へ安全に変換すること。

## 判断の順番

1. どの1本を seed 動画にするか。
2. その動画は誰の、どんな時間を助けているか。
3. 視聴者はクリック前に何を期待したか。
4. 期待と中身は一致しているか。
5. サムネ、タイトル、音楽、尺、投稿時期のどれが伸びに効いた仮説か。
6. 自分が真似てよい抽象要素は何か。
7. 真似てはいけない固有要素、危険要素は何か。
8. 次の1本で何を1つだけ変えて検証するか。

## 見るべきポイント

### 1. Seed動画性

チャンネル全体ではなく、まず1本単位で見る。

- チャンネル平均より明らかに伸びているか。
- 新しいチャンネルでも伸びたのか、既存ファンによる伸びなのか。
- 1本だけの偶然か、似た型で複数本伸びているか。
- 直近で伸びた動画か、長期で積み上がっている動画か。
- 関連動画や検索から流入しそうな設計か。

### 2. 視聴者用途

AI BGMの視聴者は、作品鑑賞者というより「生活の中で音を使う人」。分析ではペルソナ名より利用場面を優先する。

- 作業、勉強、読書、睡眠、朝の準備、夜のリラックス。
- カフェ気分、旅行気分、季節感、都会感、自然感。
- 気分を上げる、落ち着く、集中する、寂しさを埋める。
- 長時間流しっぱなしにしたいのか、雰囲気を一瞬で選びたいのか。

### 3. クリック前の約束

サムネとタイトルが、視聴者に何を約束しているかを見る。

- どんな気分になれると伝えているか。
- どんな場所・時間・季節が想像できるか。
- 音楽ジャンルがわかるか。
- 長時間BGMとして邪魔にならなそうか。
- スマホ一覧で0.5秒以内に意味が伝わるか。

### 4. サムネの分解

サムネは綺麗かどうかではなく、視聴者用途が即伝わるかを見る。

- 主役: 人物、部屋、カフェ、風景、机、窓、夜景など。
- 構図: 中央、左右余白、奥行き、視線誘導。
- 色: 暖色、寒色、低彩度、高コントラスト、季節色。
- 光: 朝、夕方、夜、ネオン、窓明かり、キャンドル。
- 密度: 情報量が多すぎないか、背景が邪魔していないか。
- 文字: 小さくても読めるか、再生時間表示にかぶらないか。
- 一覧性: チャンネル内で並んだ時に世界観が揃うか。

### 5. タイトルの分解

タイトルはSEOだけではなく、視聴者の用途を言語化する役割を持つ。

- 用途語: study, work, sleep, relax, cafe, morning, night など。
- 感情語: calm, cozy, peaceful, dreamy, nostalgic など。
- 場所語: cafe, room, city, forest, rain, ocean など。
- 時間語: morning, midnight, rainy day, autumn など。
- 言語圏: 英語圏や高RPM国に自然に届く表現か。
- 違和感: 日本語直訳っぽくないか、盛りすぎていないか。

### 6. 音楽と映像の一致

クリック後に離脱されないため、タイトル・サムネ・音楽の約束が揃っているかを見る。

- サムネが静かな夜なのに、音が明るすぎないか。
- 勉強用なのに、メロディが主張しすぎないか。
- 睡眠用なのに、急な展開や高音が多くないか。
- カフェ系なのに、画像が生活感から外れていないか。
- ループや曲間の違和感が少ないか。

### 7. 横展開可能性

伸びた動画は、すぐ派生を作れるかを見る。

- 同じ構図で、季節だけ変える。
- 同じ場所で、時間帯だけ変える。
- 同じ用途で、音楽ジャンルだけ変える。
- 同じタイトル構造で、感情語だけ変える。
- 同じ世界観で、背景小物を1つだけ変える。

横展開は「同じ視聴者が次に欲しがる動画」を作るためのもの。ベンチマーク先のコピーではなく、自分のチャンネル内の回遊を増やすために使う。

### 8. 安全性と独自性

TTPSは抽象構造を学ぶことであり、固有要素のコピーではない。

- ロゴ、チャンネル名、固有キャラ、透かしは真似ない。
- 既存アーティスト名、商標、作品名を安易にタイトルへ入れない。
- ベンチマーク画像を合成、トレース、上書きしない。
- 似せるのは構図、色、光、密度、用途訴求などの抽象要素。
- 自分の制作工程、差分、視聴者価値を説明できるように残す。

## 分析の思考原則(モデル非依存)

観点リストだけ渡すと、AIは「もっともらしい物語」を作ってしまう。どのモデルに渡す場合も、以下の思考規律を分析の前提にする。これは Fable5 が実際にベンチマーク分析をする際の判断基準を一般化したもの。

### 原則1. 観察と推測を分離し、確度を明示する

AIの最大の失敗モードは、観察(サムネに窓がある)と推測(窓が伸びの要因)を混ぜて断定すること。すべての主張に確度ラベルを付ける。

- 「観察」: URL、タイトル文字列、再生数、投稿日など、データとして見えるもの。
- 「推定」: 複数の観察から合理的に導けるもの。根拠となる観察を必ず添える。
- 「憶測」: データで裏付けられない仮説。憶測は禁止ではなく、検証すべき仮説として明示する。

### 原則2. 生存者バイアスを疑う

伸びた動画だけ見ると、無関係な共通点まで「成功要因」に見える。可能な限り「似た構成なのに伸びていない動画」を対照として探す。

- 同ジャンルで似たサムネ・タイトルの低再生動画に、その要素はあるか。
- 伸びた動画にだけあり、伸びていない動画にない差分は何か。
- 対照が見つからない場合、その要因分析は確度を1段下げる。

### 原則3. 代替説明を先に潰す

「サムネが良かったから伸びた」と結論する前に、コンテンツ以外の説明を必ず検討する。

- 投稿タイミング: 季節需要、イベント、トレンドの波に乗っただけではないか。
- チャンネル要因: 既存登録者の初速、別動画からの回遊ではないか。
- 外部流入: SNS、ブログ、プレイリスト埋め込みなど、YouTube外の要因はないか。
- アルゴリズム変動: 同時期に同ジャンル全体が伸びていないか。
- 単なる分散: 試行回数が多ければ、何本かは運だけで伸びる。

代替説明を潰せない場合は「要因不明だが観察対象」として扱い、TTPの根拠にしない。

### 原則4. 数字は正規化してから比較する

生の再生数は比較に使えない。突出度は必ず相対値で見る。

- チャンネル内比較: その動画 ÷ チャンネル中央値(平均は外れ値に弱い)。
- 時間正規化: 累計再生数ではなく、公開経過日数あたりの再生数。古い動画の積み上げと初速バズを区別する。
- 規模正規化: 登録者数に対する再生数の倍率。登録者1万で10万再生と、100万で10万再生は別物。
- ジャンル比較: そのジャンルの標準的な再生数帯を先に把握する。

### 原則5. 伸びの「形」から流入経路を推定する

伸びの時間曲線は流入経路の手がかりになる。

- 公開直後に急伸→減衰: ブラウズ面・登録者初速型。サムネ・タイトルの勝ち。
- じわじわ長期増加: 検索・関連動画型。タイトルのキーワードと用途の明確さの勝ち。
- 途中で急に段差: 外部流入かアルゴリズム再評価。コンテンツ要因と混同しない。

曲線が見えない場合は、公開日と現在再生数から少なくとも「初速型か長期型か」を推定し、どちらか明示する。

### 原則6. ファネルのどこで勝っているかを特定する

「伸びた」は1つの事象ではない。インプレッション→クリック(CTR)→視聴維持→再訪・回遊の連鎖のどこで勝ったかで、真似るべき要素が変わる。

- CTR勝ち→サムネ・タイトルを学ぶ。
- 維持率勝ち→曲構成・ループ設計・音量感を学ぶ。
- 再訪勝ち→シリーズ設計・世界観の一貫性を学ぶ。

外部からCTRや維持率は見えないため、コメント内容(「作業用に毎日聴いてる」=再訪)、高評価率、再生数と登録者の比率などの代理指標で推定し、代理であることを明示する。

### 原則7. 仮説は「効果量 × 検証コスト」で優先順位を付ける

伸び要因の仮説が複数出たら、全部真似るのではなく、次の1本で検証する1つを選ぶ。選定基準:

- 効いていた場合のインパクトが大きいか(CTR系はインパクト大)。
- 1要素だけ変えて検証できるか(交絡しない設計にできるか)。
- 結果判定に使う数字と判定期限を先に決められるか。

「次に変えるのは1つだけ」を徹底する。複数同時に変えると、何が効いたか永久にわからなくなる。

### 原則8. 見えていないデータを申告する

外部からの分析では、CTR、維持率、流入内訳、インプレッション数は見えない。AIは見えないデータを補完して断定しがちなので、分析の最後に必ず「この分析に欠けているデータ」と「そのデータが取れたら結論がどう変わりうるか」を書く。

### 原則9. 安全判定は保守側に倒す

ある要素が「真似てよい抽象要素」か「固有要素」か迷ったら、固有要素として扱う。BAN・収益化停止は非対称リスク(失うものが大きい)なので、判定の閾値は安全側に置く。独自性の上乗せが1つも思いつかない場合、その seed は採用しない。

### 原則10. 分析自体もPDCAの対象にする

分析の予測(この派生案は伸びるはず)と実際の結果を突き合わせ、外れた場合はどの原則・どの推定が誤っていたかを記録する。予測が外れ続ける観点(例: サムネ色の効果を過大評価しがち)はモデル側の癖として補正する。

## AIに渡す標準プロンプト

```text
あなたはAI BGM/作業用BGMチャンネルのベンチマーク分析担当です。

目的は、ベンチマークをコピーすることではありません。
伸びている1本の動画から、視聴者が反応した抽象パターンを抽出し、自分のチャンネルに安全に応用することです。

分析にあたっての思考規律:
- すべての主張に確度ラベルを付ける: [観察] データとして見えるもの / [推定] 観察から導いたもの(根拠を添える) / [憶測] 裏付けのない仮説。
- 「伸びた要因」を挙げる前に、代替説明(投稿タイミング、既存登録者、外部流入、ジャンル全体の波、単なる運)を検討し、潰せないものは要因扱いしない。
- 再生数は生の値で比較しない。チャンネル中央値との比率、公開経過日数あたり、登録者数との倍率で見る。
- 伸び方が初速型(サムネ・タイトル勝ち)か長期型(検索・関連勝ち)かを推定し、ファネル(インプレッション→CTR→維持→再訪)のどこで勝った動画かを明示する。
- 迷った要素は「真似てはいけない固有要素」側に倒す。

以下の観点で分析してください。

1. seed動画候補
- チャンネル全体ではなく、特に分析対象にすべき1本を選ぶ。
- なぜその動画がseedになるのかを説明する。

2. 視聴者用途
- 視聴者がどんな場面で使う動画かを推定する。
- 作業、勉強、睡眠、リラックス、カフェ、季節感、旅行気分などの観点で見る。

3. クリック前の約束
- サムネとタイトルが、視聴者にどんな気分・用途・音楽を約束しているかを分解する。

4. サムネ分析
- 主役、構図、色、光、余白、文字、スマホ視認性、チャンネル一覧での統一感を見る。

5. タイトル分析
- 用途語、感情語、場所語、時間語、言語圏への自然さを見る。

6. 音楽/映像一致
- タイトル、サムネ、音楽が同じ視聴シーンを約束しているかを見る。

7. 横展開案
- 伸びた要素を1つだけ変える派生案を3つ出す。

8. PDCA仮説
- 次の1投稿で検証する要素を1つだけ選ぶ(複数同時に変えない)。
- 効果量と検証しやすさで選定理由を説明する。
- 成功判断に見るべき数字・反応と、判定する期限を書く。

9. 安全性
- 真似てよい抽象要素と、真似てはいけない固有要素を分ける。
- 著作権、商標、既存アーティスト名、固有キャラ、再利用コンテンツのリスクを指摘する。

10. 分析の限界
- この分析で見えていないデータ(CTR、維持率、流入内訳など)を列挙する。
- そのデータが取れた場合、どの結論が変わりうるかを書く。

出力形式:
- seed動画候補(選定理由と突出度の根拠つき):
- 伸び方の型(初速型/長期型)とファネル上の勝ち所:
- 視聴者用途:
- クリック前の約束:
- 効いていそうなサムネ要素(確度ラベルつき):
- 効いていそうなタイトル要素(確度ラベルつき):
- 検討した代替説明と潰せたかどうか:
- 真似てよい抽象要素:
- 真似てはいけない固有要素:
- 横展開案3つ:
- 次のPDCA仮説(変える1要素・成功指標・判定期限):
- この分析に欠けているデータ:
```

## Fable5などの分析ツールで確認したいデータ

- 動画URL、タイトル、サムネ、投稿日。
- 再生数、いいね数、コメント数。
- チャンネル登録者数、チャンネル開設時期。
- チャンネル平均に対する当該動画の突出度。
- 直近30日で伸びているか。
- 類似サムネ/類似タイトルの複数動画が伸びているか。
- コメントから見える用途、国、言語、感情。
- 自チャンネルで横展開した時の初速、CTR、平均視聴時間、関連流入。
