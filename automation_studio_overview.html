Automation Studio — 仕様とできること

YouTube の BGM チャンネル向けに、動画づくりのほぼ全工程を自動化する道具です。 「音楽をつくる → 表紙画像をつくる → 動画編集ソフトに並べる → 書き出す → 説明文を書く → YouTube に投稿する」までを、1 つの管理画面からまとめて動かせます。

管理画面 パソコン上で動くアプリ 作業工程 11 段階 考える仕事 AI が担当 更新 2026-06-08 時点
このページの読み方。 前半「はじめての方へ」は、専門知識がなくても読めるよう、できるだけやさしい言葉で書いています。 後半「くわしい仕様」は、プログラムの中身を知りたい開発者向けの技術解説です。 むずかしい言葉が出てきたら、いちばん下の用語集を見てください。
はじめての方へ

1ひとことで言うと

これは「YouTube に上げる音楽動画を、なるべく自分の手を動かさずに作るための工場」のような道具です。 ふだん人が時間をかけてやっていた作業を、ボタン 1 つ、あるいは話しかけるだけで、機械と AI が代わりにやってくれます。

たとえると「自動の動画工場」

材料（音楽・画像・文章）を流し込むと、ベルトコンベアのように各工程を通って、最後に YouTube 投稿まで完成品が出てきます。

「考える仕事」は AI 担当

曲のタイトル、動画の説明文、サムネイルに入れる言葉など、頭を使う部分は AI（Claude／Codex）が考えます。

「手を動かす仕事」は機械担当

音楽生成サイトの操作、動画編集ソフトへの配置、YouTube への投稿は、プログラムが自動で操作します。

はじめての方へ

2できること

この道具でできることを、ふだんの言葉で並べます。すべて自動、または半自動です。

  • 音楽を自動で作る音楽生成 AI「SUNO」に、1 本の動画ぶん（例：20 曲）をまとめて作らせます。曲名や雰囲気の指定も AI が考えます。
  • 音楽をきれいに整える曲に英語のタイトルを付け直し、終わりをやさしくフェードアウト、音量をそろえる、といった仕上げを自動で行います。
  • サムネイル（動画の表紙画像）を作るライバルの人気動画を参考に背景画像を生成し、その上に文字をのせて、表紙を 2 種類つくります。
  • 動画編集ソフトに素材を自動で並べるAdobe Premiere Pro を自動で操作し、音楽・字幕・画像を時間どおりに並べて、3 時間などの長い動画を組み立てます。
  • 動画ファイル（MP4）を書き出す編集が終わったら、投稿用の動画ファイルを自動で書き出します。出来上がりも自動でチェックします。
  • タイトル・説明文・タグを AI に考えてもらう動画のタイトル候補を 5 つ、説明文、タグを AI が提案。さらに他の言語へ自動で翻訳もできます。
  • YouTube へ自動で投稿する限定公開・一般公開・予約投稿に対応。投稿が終わると「投稿済み」の印が自動で付きます。
  • ライバルを分析して作戦を立てる伸びている競合チャンネルを調べ、「まねする点・避ける点・自分流に進化させる点」の 3 つに分けて提案します。
  • 外出先のスマホからも操作できる専用の通り道（Cloudflare Tunnel）を使って、外からでも同じ管理画面を開けます。
  • 時間がきたら自動でおまかせ運転（準備中）「毎週この時間に新しい動画を作る」といった無人運転のしくみも用意されています（最終の起動スイッチは安全のため保留中）。

33 つの使い方（操作の入口）

この道具には「動かし方」が 3 通りあります。どの方法を使っても、最終的には同じ 1 つの中枢プログラムが動きます。やさしい順に紹介します。

① 画面のボタンを押す

パソコンでアプリを開き、動画ごとに並んだボタン（音楽を作る・サムネを作る・投稿する など）を押すだけ。いちばん分かりやすい方法です。

外出先のスマホからも同じ画面を開けます

② AI に日本語で頼む

「vol.78 を作って」「競合を分析して」のように、AI アシスタント（Claude Code）へ日本語で頼むと、必要な操作に翻訳して実行してくれます。

ボタンを探さなくても、話しかけるだけで動きます

③ 時間がきたら自動で（準備中）

「毎週この曜日に新しい動画を作る」など、あらかじめ予定を組んでおくと、人がいなくても勝手に作業を進めます。

安全のため、最後の投稿だけは人の確認が必要な設計

ポイント「②の AI に頼む」で出てくる Claude Code と、「サムネの文字や曲名を考える AI」は、同じ Claude でも役割が違います。 前者はあなたの代わりに操作する秘書、後者は中身を考える職人です。くわしくは次の章で説明します。
はじめての方へ

4AI（Claude と Codex）の役割

この道具の「考える部分」はすべて AI に任せています。使う AI は 2 つあり、片方が使えない時にもう片方へ自動でバトンタッチするので、途中で止まりにくいのが特長です。

ふだんの担当：Claude

曲のタイトル、説明文、サムネの文字、ライバル分析など、頭を使う仕事はまず Claude という AI が担当します。

パソコンに入った Claude を呼び出して使うので、特別な契約キーは不要です

控えの担当：Codex

Claude が混雑や上限・エラーで使えない時は、自動で Codex という別の AI に切り替わります。だから作業が止まりにくいのです。

Codex を使うには、事前にログインしておく必要があります

かんたんに言うと「Claude が手いっぱいなら Codex が代わりに考える」。 この自動の切り替えがあるおかげで、夜間の無人運転でも作業が途切れにくくなっています。
はじめての方へ

5動画ができるまでの流れ（11 段階）

1 本の動画は、次の順番で組み立てられます。それぞれの段階で「できたかどうか」は、フォルダの中にファイルが出来ているかで判断します。

音楽生成SUNO→ 仕上げ名前・音量→ 背景画像AIが生成→ サムネ合成文字のせ→ 編集ソフト配置Premiere→ 書き出しMP4→ 自動チェックQA→ 説明文作成AI→ 多言語化翻訳→ サムネ最終予備経路→ 投稿YouTube

自動おまかせ運転（使い方③）のときだけ、いちばん前に「作戦立て」が 1 段階増えて 12 段階になります。 また、サムネイルは通常「背景画像 → 文字のせ」の順で作り、うまくいかない時だけ「サムネ最終（予備経路）」で AI が直接つくり直します。

7Claude / Codex 切替回路（フォールバック）

システム内の「AI に考えさせる処理」は、必ず app_llm_runner.py の run_llm() / run_llm_vision() を通ります。文章生成も画像理解（Vision）もここに集約され、全モジュール（約 26 箇所）の AI 呼び出しが共通化されています。新しく作る処理も run_llm 必須で、直接 claude -p を叩くのは禁止です。

呼び出し元（音楽 / 説明文 / 背景 / 分析 / サムネ文字 / 画像採点 …） │ run_llm(プロンプト) / run_llm_vision(プロンプト, 画像) ▼ ┌──────────────────────────────────────────────────────────┐ │ ① まず Claude CLI を試す │ │ claude -p <プロンプト> [--allowedTools Read] [--add-dir …]│ │ 成功（終了コード0かつ出力あり）→ そのまま返す │ └───────────────┬──────────────────────────────────────────┘ │ 失敗（エラー / 空 / タイムアウト / コマンド無し ＝上限など） ▼ ┌──────────────────────────────────────────────────────────┐ │ ② Codex CLI に自動で切り替え │ │ codex exec --skip-git-repo-check │ │ --dangerously-bypass-approvals-and-sandbox │ │ [-i <画像> …] -o <最終メッセージ保存先> <プロンプト> │ │ 成功 → 「Codex で成功」をログに残して返す │ └───────────────┬──────────────────────────────────────────┘ │ 両方とも失敗 ▼ LLMError を送出（401系なら「codex login を」と案内付き）
第一候補 Claude CLI

claude -p "<プロンプト>" を子プロセスで起動。API キー不要でローカル認証完結。画像理解のときは画像の親フォルダを --add-dir で許可し --allowedTools Read を付けます。

コマンドのパスは suno_config.json の claude_cli で解決

控え Codex CLI

Claude が上限・エラー・タイムアウト・未検出になると自動で codex exec に引き継ぎます。-o の最終メッセージファイルを最優先で読み（無ければ標準出力）、画像は -i で渡します。

事前に codex login が必要（未認証だと 401 で両方失敗・認証は失効しやすい）

補足この切り替えは既定で有効。環境変数 APP_LLM_FALLBACK=0 で無効化できます（その場合 Claude 失敗で即エラー）。切り替えは一方向（Claude → Codex のみ）です。

9プログラム解説 — 中枢

操作の入口を受け、各工程プログラムを呼び出す司令塔。巨大化した app.py（旧 12,649 行）を D9 リファクタで core + routers に段階分割しました。

app.py 7,921 行

FastAPI 組み立て役 + 実行基盤。ミドルウェア / 起動フック / 各種ワーカー基盤（書き出し監視・順番待ち・スケジューラ・公開ゲート）。3 つの入口がすべてここに集約する単一頭脳。

app_core.py 1,432 行

共通部品（D9 抽出）。パス・設定の定数、設定ローダ、チャンネル別設定、認証ヘルパ、タスク／子プロセス／YouTube キューのヘルパなど。

routers/

機能別の窓口（APIRouter）。benchmark.py（ベンチ4軸）/ images.py（背景・サムネ・承認・series）/ premiere_photoshop.py（JSX・UXP・文字）/ youtube.py（投稿・履歴・説明文・通知）。

web/static/index.html

管理画面 UI（JavaScript も同梱・ビルド不要）。画面側の工程判定は WORKFLOW_STEPS 定数が真実源。

10プログラム解説 — 流れの統括・自動運転

app_pipeline.py 2,940 行

一括パイプラインの統括。STEPS が工程の真実源。--only / --from / --dry-run / --from-benchmark / --via-api に対応。QA・翻訳・作戦立てなどの工程関数も内包。

「全部やって」「Premiere からやり直して」の実体

app_orchestrator.py 650 行

自動運転の司令塔（Phase 4）。StageWorker / QAWorker / PlanWorker。ベンチ起点で作戦を立て工程を順送り。書き出し〜サムネを自動投入し、投稿は手動ゲート。連続 3 失敗でブレーカー作動。

app_render_queue.py 395 行

Premiere / Media Encoder は同時に 1 つしか動かせない、という物理制約を吸収する SQLite ベースの順番待ち。複数チャンネル並列運用の中核。

app_run_ledger.py 492 行

中央の実行台帳（SQLite）。これまでメモリ最大 50 件だった履歴を一元・永続記録。台帳 UI のデータ源。

app_token_health.py 294 行

認証の有効期限チェッカ。チャンネル別に点検し、期限が近いものを Discord で「次回ログインが必要」と事前通知。

app_image_eval_loop.py 315 行

自己評価つき画像生成ループの核。生成 → ベンチ比較で採点 → 合格まで弱点を還流して再生成（外側ループは計画段階）。

現状自動運転のバックエンド（Phase 0-5）と管制 UI（U1-U6）は全実装済みですが、 app.py への最終統合（定時の合図でスケジューラを回し無人稼働を開始する配線）は「要 GO」で保留中です。 自走 ON/OFF トグルは現状保存のみで、実際の無人稼働はまだ始まっていません。

11プログラム解説 — 各工程の担当

パイプラインや API から子プロセスとして呼ばれる「実際に手を動かす」層。単体のコマンドとしても実行できます。

suno_auto_create.py

Playwright で suno.com を操作して楽曲生成。Workspace 管理（vol 別）/ 生成モード 3 種 / 一括生成（--batch で N 曲を 1 往復）/ 一括ダウンロード（fetch インターセプタ方式）。

注意：Bot 判定対策で APP_KEEP_BROWSER=1 + interval 90 を既定推奨

app_process_tracks.py 850 行

楽曲後処理。サムネ → ペルソナの順で英語タイトルを提案しリネーム、ffmpeg で無音トリム + フェードアウト + 音量正規化（LUFS）。--rename-only あり。公開前に SUNO の透かし除去 + ID3 タグ付与。

app_premiere.py 906 行

Pymiere で .prproj を開き JSX 自動配置（音声/字幕/画像）、実測タイムから正確な SRT を Python 生成。--export で書き出しキュー投入、--images-only で画像のみ再配置。

注意：JSX は JSX Launcher 拡張経由のみ（AppleScript 不可）

app_photoshop.py 1,017 行

Photoshop 連携ラッパ。新版（PS 2026）は UXP のファイル連携、旧版は AppleScript。背景 + シーンテキストで vol{N}.jpg + サムネイル.jpg を 2 枚出し。

app_youtube.py 919 行

YouTube 投稿。OAuth → MP4 / サムネ / 説明 / タグ / タイトルを保存ファイルから自動読込 → videos().insert()。成功後 youtube_upload.json を書出。--auth-only で再認証。

app_image_prompt.py / codex_imagegen.py

背景画像。app_image_prompt がベンチ分析から動的プロンプトを構築し、codex_imagegen が OpenAI Image API / Codex CLI で（参照画像対応・並列）生成。

app_channel_thumbnail.py 278 行

チャンネル横断・AI サムネ一括生成（ピュアロジック層）。PSD 合成が使えない時の予備経路。{vol}/Image/ に一括生成 + 採点。

claude_proposer.py / scene_text_generator.py

前者は動画メタ提案（titles / description / tags、gather_context() でフォルダから文脈を自動収集）。後者は AI 画像を見て焼き込む英大文字フレーズを生成（per-channel scene_text_* 設定準拠）。

12プログラム解説 — 分析・調査

13skills とは何か

skills/ フォルダは、プログラム本体ではなく AI アシスタント（Claude Code＝使い方②の秘書役）のための手順書集です。中身は人間も読める Markdown 文書で、各機能を「どんなコマンドで・どんな順番で・どんな注意点に気をつけて」実行するかが書かれています。

役割：AI の取扱説明書

あなたが「サムネを作って」と頼むと、AI アシスタントはまず該当する手順書（app-thumbnail.md）を読み、正しいやり方・落とし穴を理解してから作業します。だから指示がざっくりでも、正しい手順で実行されます。

位置づけ：実行はしない・案内する

skills 自体は動くプログラムではありません。実際に動くのは app_*.py などのプログラムです。skills は「どのプログラムをどう呼ぶか」を AI に教えるナレッジ層で、§3 の「使い方②」を支えています。

16注意点・つまずきやすい所（実戦知見）

サーバー / 実行
  • uvicorn は自動リロード無し → 変更後は bash start.sh で再起動
  • AI 工程（meta / localization）を --via-api 実行すると 10 秒タイムアウトで必ず失敗 → CLI 直で実行
  • premiere / export の「完了」ログは早期誤判定あり → MP4 サイズ安定 + ffprobe で裏取り
  • JSX は JSX Launcher 拡張経由のみ（AppleScript 不可）
SUNO（音楽生成）
  • Bot 判定対策で APP_KEEP_BROWSER=1 + interval 90 を既定に
  • 生成前に prompt 本文をユーザーに見せて合意を取る（ジャンル違い事故の防止）
  • 複数 vol は workspace を vol 別にして混在ダウンロードを防ぐ
  • 生成リクエスト送信 ≠ レンダリング完了（DL 前に待機 / cache miss は再試行）
Premiere / Media Encoder
  • batch.xml の古いジョブで別 vol を誤書き出し（最重要）
  • AME 強制終了で Premiere のモデルが壊れるリスク
  • Premiere 再起動後は AME も再起動しないと描画が始まらない
  • PS 2026 は字幕トラックを削除できず積み上がる
AI / Codex / 共有ドライブ
  • Codex は codex login 必須・認証が失効しやすい。切替は Claude → Codex の一方向
  • YouTube quota はローカル追跡が保守的。実 API は深夜 0 時 PT にリセット済みのことあり
  • Google ドライブ上は Bash 出力が壊れることがある → 検証は /tmp 経由、git はパス限定
  • 認証の有効期限切れを検出したら再ログインが必要

17用語集

API（エーピーアイ）
プログラム同士が情報をやり取りするための窓口・約束ごと。「YouTube に投稿する窓口」など。
CLI（シーエルアイ）
コマンド入力でプログラムを動かす方式。ここでは claude -p … のように Claude を文字コマンドで呼び出すこと。
子プロセス / subprocess
あるプログラムが、別の小さなプログラムを「呼び出して実行」すること。中枢が各工程の道具を呼ぶ仕組み。
フォールバック
本命がダメな時に控えへ自動で切り替えること。ここでは Claude → Codex の切替。
パイプライン
作業を決まった順番でつなげて一気に流す仕組み。ここでは 11 工程の流れ作業。
ベンチマーク
比較・目標にする基準。ここでは参考にするライバルチャンネルやその分析。
サムネイル
動画の表紙画像。クリックされるかどうかを大きく左右する。
メタ（メタデータ）
動画のタイトル・説明文・タグなど、動画に付ける情報。
OAuth（オーオース）
パスワードを直接渡さずにサービス連携する認証方式。YouTube 投稿で利用。
SQLite（エスキューライト）
1 ファイルで動く軽量データベース。ここでは順番待ちと履歴台帳に使用。
スケジューラ
決めた時刻・曜日に自動で処理を起動するしくみ（APScheduler）。
JSX
Adobe 製品を自動操作するためのスクリプト。Premiere の自動配置で使用。
