# app-imitate-evolve: 徹底パクリ進化分析

「パクって終わり」ではなく **「徹底的にパクり、進化する」** 戦略フレームワーク。
ベンチマーク先の TOP 動画 + 自チャンネルのペルソナから、
`✓ パクる要素` / `✗ 避ける要素` / `+ 進化させる要素` の 3 軸で Claude に提案させる。

## 設計思想

BGM / インスト音楽チャンネルでは、視聴者のニーズ（睡眠/作業/カフェ雰囲気）が明確で、
勝ちパターンが既存チャンネルに確立されている。**完全オリジナルを目指すより、勝ちパターンを踏襲しつつ、
自チャンネルのコンセプト（ペルソナ）で差別化する方が効率的**。

ただし:
- **何をパクり、何を避け、何を進化させるか**の判断が難しい
- 毎回ぶれるとチャンネルの一貫性が失われる（コンセプト崩壊）
- そこで Claude CLI にペルソナ + ベンチマーク TOP 動画を渡して整理させる

## 3 軸の定義

| 軸 | 意味 | 例 |
|---|------|----|
| ✓ **パクる要素** | 型・構成・タイトルパターン・投稿時刻など、**視聴者を惹きつける勝ちパターン** | 「時刻 + 場所 + 雰囲気」のタイトル構成 / 深夜〜早朝の投稿時刻 / 1時間超の長尺 |
| ✗ **避ける要素** | 既出すぎて差別化できない、ペルソナと合わない、視聴者疲労が出ている部分 | ありきたりな Lofi Girl 風サムネ / 英語タイトルの乱立 / BPM 70 のお決まり |
| + **進化させる要素** | 自チャンネルペルソナの強みでベンチマークの弱点を補う独自要素 | 「東京のペントハウス」など具体的な場所性 / ジャズハウス寄りのレアなニッチ / 質感へのこだわり |

## UI フロー

1. 動画詳細 → 「メタ」タブ
2. 「📊 競合分析から提案」セクション → **「🧬 徹底パクリ進化を提案」** ボタン
3. 約 60 秒で Claude が 3 軸 + サマリを生成
4. 結果は `--accent-success`（✓）/ `--accent-danger`（✗）/ `--accent-info`（+）で色分け表示

```
🧬 徹底パクリ進化分析
┌──────────────────────────────┐
│ [サマリ] 日本語 1〜2 文の総合戦略 │
├──────────────────────────────┤
│ ✓ パクる要素（型・構成）        │
│   - 要素 1                    │
│   - 要素 2                    │
├──────────────────────────────┤
│ ✗ 避ける要素（差別化）          │
│   - 要素 1                    │
├──────────────────────────────┤
│ + 進化させる要素（ペルソナ × 弱点）│
│   - 要素 1                    │
└──────────────────────────────┘
```

## API

### POST `/api/videos/{video_name}/suggest-imitate-evolve`

内部的に以下を組み合わせて Claude に投げる:
1. `dashboard_config.persona`（必須、未設定ならエラー）
2. `/api/analysis/benchmark-videos` の結果（ピン留め or hot 上位 5 ch × TOP3 + RECENT2）
3. Sheet1 由来の TOP 動画タイトル / 視聴数 / 投稿日

返却スキーマ:
```json
{
  "imitate": ["型・構成の提案", ...],
  "avoid": ["避けるべき要素", ...],
  "evolve": ["進化ポイント", ...],
  "summary": "日本語 1〜2 文の総合戦略"
}
```

### プロンプトカスタマイズ

**マスター設定 → プロンプト管理 → `imitate_evolve`** で上書き可能。
既定プロンプトは [Python/app.py](../Python/app.py) の `api_suggest_imitate_evolve` 内に記述。
プレースホルダー不使用（引数を文字列連結で構築）なので、完全日本語で書き換え自由。

## ベンチマーク参照サイドパネル

メタタブ下部のアコーディオン **「📊 ベンチマーク参照」** で、AI 提案の**根拠データ**を確認できる。
3 つのサブセクション:

### 🎯 適用中ベンチマーク
ピン留め（`benchmark_config.json`）がある場合はそれ、無ければ hot 上位 5 ch。
各チャンネルごとに:
- 登録者数
- TOP 動画 3 件（サムネ + タイトル + 視聴数 + 投稿日）

### 📅 投稿時刻パターン
対象チャンネルの TOP 動画の `publishedAt` を「曜日 × 時刻」の 7×24 ヒートマップで可視化。
強度は `--accent-primary` の opacity 0〜1。

→ **何曜日・何時に投稿すべきかの意思決定材料**。

### 🏷 タイトル頻出ワード TOP15
Sheet1 の TOP 動画タイトルから英単語 + 日本語 2文字以上を集計。
一般語（the/and/for/music/bgm/vol/playlist 等）は除外。

→ **視聴者が検索しそうなワード**の傾向把握。

### API

| メソッド | パス | 用途 |
|---------|------|------|
| GET | `/api/analysis/benchmark-videos` | ピン留め対象の TOP/最新動画 + メタ |
| GET | `/api/analysis/posting-times` | 曜日×時刻ヒートマップ（計 25〜件程度） |
| GET | `/api/analysis/tag-frequency?top_n=15` | タイトル単語頻度 |

### 対象チャンネルの選定

`_benchmark_target_channels()`:
1. `benchmark_config.pinned_names` があればそれを最大 8 ch
2. 無ければ `api_hot_channels(top_n=5)` の上位 5 ch をフォールバック

これにより「ベンチマーク対象が空だと何も出ない」を避ける。

### Sheet1 マッチング

Sheet1 の `channel.title` と Sheet2 の `channel_name` で突き合わせ:
1. 完全一致 → `detail_by_name[name]`
2. 無ければ `match_channels()` で fuzzy（`SequenceMatcher` ratio > 0.8）
3. それでも見つからなければ `found: false` で返却（UI では「Sheet1 で見つからず」表示）

## コンセプトのブレ防止

「徹底パクリ進化」を繰り返し使っても、ペルソナを毎回渡す（`dashboard_config.persona`）ので、
**チャンネルの世界観**（例: orzz. = 「デジタル空間のプライベート・ラウンジ」「質感への執着」）が軸として効き続ける。

ペルソナを丁寧に書いておくことで:
- 「avoid」に「ペルソナと合わない要素」が自動的に入る
- 「evolve」に「ペルソナの強み × 競合の弱点」が提案される

つまり**ペルソナが判断基準そのもの**として機能する。

## 運用の推奨フロー

1. 動画フォルダ作成後、メタタブの「📊 ベンチマーク参照」を展開して**対象チャンネルの投稿時刻・頻出ワード**を確認
2. 「🧬 徹底パクリ進化を提案」で**戦略**を受け取る
3. 「🚀 一気通貫提案」で**具体的な SUNO プロンプト / タイトル / 説明 / タグ**を生成（Flow サムネプロンプト提案は D8 で撤去）
4. 「📒 A/B ログ」でその時の条件を記録
5. 7 日後に再生数・登録者増を入力 → **どの戦略が効いたかを定量化**

## トラブルシュート

| 現象 | 原因 | 対処 |
|------|------|------|
| 「ペルソナ未設定」エラー | `dashboard_config.persona` が空 | 基本設定 → ペルソナを入力 |
| 「ベンチマーク対象が空」 | ピン留めも hot ranking も取れない | スプレッドシート URL を設定タブで入力 → 「日本語で再分析」 |
| 結果が英語 | 旧プロンプトのキャッシュ | マスター設定の `imitate_evolve` プロンプトは日本語固定なので発生しないはず。他の analysis が英語なら [app-competitor-spreadsheet.md](./app-competitor-spreadsheet.md) 参照 |
| 投稿時刻ヒートマップが全部 0 | Sheet1 の `publish_datetime` 列（E列相当）が空 | Sheet1 側のデータ整備が必要 |
