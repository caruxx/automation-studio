# Automation Studio Design System

メルカリのデザインシステム再構築の思想、およびデジタル庁デザインシステムβ版のコンポーネント体系を参考に、Automation Studioのデザインガイドラインを定義する。

## 設計原則

### 1. Semantic（意味的分類）
- コンポーネントは見た目ではなく**役割・意味**で分類する
- 例: 「青いボタン」ではなく「プライマリアクション」「デストラクティブアクション」

### 2. Consistency（一貫性）
- 同じ操作は常に同じ見た目・動作をする
- スペーシング・フォントサイズ・カラーは必ずトークンを使う

### 3. Simplicity（シンプルさ）
- 過度な汎用化を避け、必要十分なバリエーションのみ提供
- 1画面1目的 — 画面に複数の主目的を詰め込まない

### 4. Feedback（フィードバック）
- 全ての操作に視覚的フィードバックを返す（ローディング、成功、エラー）
- リアルタイム状態は常にヘッダーに表示

### 5. Accessibility First（アクセシビリティ最優先）
- JIS X 8341-3:2016 適合レベルAA、WCAG 2.2 の AおよびAA達成基準を最低要件とする
- コントラスト比（テキスト 4.5:1 以上、非テキスト 3:1 以上）、ターゲットサイズ（24×24 CSS px 以上）、キーボード操作、スクリーンリーダー対応を必須

## コンポーネント設計

コンポーネント体系は **アプリ固有のAtomic Design 分類** と **デジタル庁デザインシステム（DADS）のコンポーネント全42種** の2軸で整理する。独自コンポーネントは本アプリの業務特性（ライブストリーム管理・YouTube運用・SUNO生成等）に特化したもの。

### アプリ固有コンポーネント（Atomic Design）

#### Atoms（原子）

| コンポーネント | 用途 | DADS対応 |
|---|---|---|
| Button | プライマリ / セカンダリ / デストラクティブ / ゴースト | ボタン |
| Input | テキスト入力 | インプットテキスト |
| Textarea | 複数行テキスト | テキストエリア |
| Select | ドロップダウン選択 | セレクトボックス |
| Badge | ステータス表示（成功/警告/エラー/情報） | チップラベル |
| StatusDot | 実行状態のドット表示 | —（アプリ固有） |
| Toggle | ON/OFF スイッチ | —（アプリ固有） |
| IconButton | アイコンのみのボタン | ボタン（icon-only） |

#### Molecules（分子）

| コンポーネント | 用途 | DADS対応 |
|---|---|---|
| FormField | ラベル + 入力 + ヘルプテキスト | インプットテキスト＋ラベル |

| VideoCard | 動画フォルダのステータスカード | カード |

| Toast | フィードバック通知（画面右下） | ノティフィケーションバナー |

#### Organisms（有機体）

| コンポーネント | 用途 |
|---|---|
| SunoPanel | SUNO設定 + 実行コントロール |
| VideoList | 動画フォルダ一覧 |
| UploadForm | YouTubeアップロードフォーム |
| SettingsForm | 設定フォーム |

## デジタル庁デザインシステム（DADS）コンポーネント全42種

Automation Studioで採用可能な標準コンポーネント群。必要に応じてダークテーマに合わせて再スタイルする。

### 📂 入力系コンポーネント

#### インプットテキスト
- **用途**: 名前や電話番号など、**1行以内のテキスト入力**
- **実装**: `<input type="text">` 等に対応
- **使用例**: API Key、タイトル、ファイル名
- **アクセシビリティ**: ラベルを必ず関連付け、エラー時は `aria-invalid="true"` + エラーメッセージ

#### テキストエリア
- **用途**: **自由記述テキスト（複数行）**
- **実装**: `<textarea>`
- **使用例**: 動画説明文、ログメモ、タグ入力

#### セレクトボックス
- **用途**: 複数の選択肢を提供する**フォームコントロール**
- **実装**: `<select>` 要素ベース
- **使用例**: チャンネル選択、プロバイダー選択、公開設定

#### チェックボックス
- **用途**: **複数項目**の中から**複数の選択肢**を選ぶ
- **アクセシビリティ**: 24×24 CSS px以上のターゲットサイズ確保
- **使用例**: 複数動画の一括選択、通知ON/OFF項目

#### ラジオボタン
- **用途**: ユーザーが選択肢の中から**1つだけ**を選択
- **使用例**: 公開範囲（公開/限定公開/非公開）、アップロード先

#### 日付ピッカー／カレンダー
- **用途**: **日付選択**のフォームコントロール。入力フィールド + カレンダーで構成
- **使用例**: 予約投稿日時、ログフィルタの期間指定

#### ファイルアップロード／ドロップエリア
- **用途**: `<input type="file">` に対応、**単体/複数ファイル**のアップロード
- **使用例**: サムネイル画像、音源、動画ファイルのドロップ

#### ランゲージセレクター
- **用途**: ユーザーが**表示優先言語を選択**するUI要素
- **使用例**: 多言語ダッシュボード対応時（将来拡張）

---

### 📂 アクション系コンポーネント

#### ボタン
- **用途**: 主に**アクション実行**または**ページ遷移**のトリガー
- **本アプリのバリエーション**:
  - プライマリ（メインアクション）
  - セカンダリ（サブアクション）
  - デストラクティブ（削除・停止）
  - ゴースト（中立的なサブアクション）
- **アクセシビリティ**: 44×44 CSS px 以上のターゲットサイズ（アイコン単体時）、フォーカスインジケーター必須

#### ハンバーガーメニューボタン
- **用途**: 画面スペース制限のあるモバイルデバイスで、**モバイルメニューを表示するトリガー**
- **使用例**: スマートフォン画面でのタブバーの代替

#### スクロールトップボタン
- **用途**: ページ先頭へ**迅速に移動**
- **使用例**: 長いログ表示画面、動画リスト画面

---

### 📂 ナビゲーション系コンポーネント

#### グローバルメニュー
- **用途**: 利用者を目的のページへ**誘導**し、コンテンツを分かりやすく表示
- **使用例**: [SUNO] [Videos] [YouTube] [通知] [設定] のメインタブ

#### ヘッダーコンテナ
- **用途**: ヘッダーに含まれる**多様なコンポーネントを内包**するコンテナ
- **使用例**: Logo + Channel Badge + Status + Clock の統合

#### ボトムナビゲーション
- **用途**: **2〜5個の目的地**へのアクセスを可能にする
- **使用例**: モバイルビューでの主要タブ

#### パンくずリスト
- **用途**: ウェブサイトの**階層内での現在位置**を表示
- **使用例**: `ダッシュボード > 動画 > vol65 > 編集`

#### ページナビゲーション
- **用途**: 前ページ・次ページなど**複数ページ間の遷移**。コンテンツ下部に設置
- **使用例**: ログのページング、動画リストのページング

#### ステップナビゲーション
- **用途**: **一連の手続きを論理的なステップで提示**。完了までの作業俯瞰、進行状況把握
- **使用例**: 動画アップロードのウィザード（情報入力 → サムネ設定 → 公開設定 → 確認）

#### メガメニュー
- **用途**: グローバルメニューから呼び出される、**メニューリスト以外のコンテンツも内包**できる大型ドロップダウン
- **使用例**: YouTubeタブのサブメニュー群（動画・ショート・ライブ・再生リスト）

#### メニューリスト
- **用途**: メニューアイテムを束ねたコンポーネント。**ローカルメニュー**として配置したり、ドロップダウン/メガメニュー内に内包
- **使用例**: カード内のアクションメニュー

#### メニューリストボックス
- **用途**: テキストラベル + 開閉アイコンのオープナーで**開閉し、メニューリストを内包**
- **使用例**: フィルタのグループ化選択

#### モバイルメニュー
- **用途**: タッチスクリーンのタップ操作を考慮した**モバイル用メニュー**。ハンバーガーメニューボタン・ドロワーと併用
- **使用例**: モバイル画面のフルスクリーンメニュー

#### ユーティリティリンク
- **用途**: **横並びリンクリスト**に近いが、よりコンパクト
- **使用例**: フッターのプライバシーポリシー・利用規約リンク

---

### 📂 情報表示系コンポーネント

#### カード
- **用途**: **単一の主題に関するコンテンツをまとめて表示**するコンテナ
- **使用例**: VideoCard、StatCard、設定カード

#### 箇条書きリスト
- **用途**: **1つ以上の項目を列挙**
- **使用例**: 機能一覧、注意事項

#### 説明リスト
- **用途**: 「説明したいこと」 +「その説明文」のペアが複数連なるコンテンツ
- **実装**: `<dl><dt><dd>`
- **使用例**: 動画メタデータ表示（タイトル: ○○、再生時間: ○○）

#### テーブル／データテーブル
- **用途**: データや情報を**行と列の組み合わせで構造化**した表組
- **使用例**: 動画一覧、アップロード履歴、ログテーブル

#### テーブルコントロール
- **用途**: データテーブルで**表示する行・列を絞り込み**、制御操作を行うコントロール群
- **使用例**: 動画リストのフィルター・ソート・表示列選択

#### リソースリスト
- **用途**: **共通した複数の任意情報**で構成されたオブジェクトのリストを表示
- **使用例**: 動画アセット一覧、音源アセット一覧

#### チップタグ
- **用途**: アイテム化した任意の情報を**表示・削除しやすく**する要素。属性やユーザー情報を並べたり削除したりする
- **使用例**: YouTubeタグ入力、キーワードフィルタ

#### チップラベル
- **用途**: **状態や状況を示すキーワード**を表示。情報リストやテーブル各行のステータス表示
- **使用例**: 「公開中」「ドラフト」「処理中」「エラー」バッジ

#### 引用ブロック
- **用途**: **段落単位での引用**を表し、これ自体も段落となる
- **実装**: `<blockquote>`
- **使用例**: リリースノート内のユーザー要望抜粋

---

### 📂 画像・メディア系コンポーネント

#### イメージスライダー
- **用途**: **ギャラリーコンテンツ**のように複数のイメージを切り替えて閲覧
- **使用例**: サムネイル候補のスライド表示

#### カルーセル
- **用途**: 複数のスライドを順番に見る。アイキャッチとしても使われるが、**使用に注意が必要**
- **注意**: 自動再生、ユーザー制御、アクセシビリティ配慮が必須
- **使用例**: ダッシュボードトップのお知らせ（使用は慎重に）

---

### 📂 通知・フィードバック系コンポーネント

#### 緊急時バナー
- **用途**: **本来のコミュニケーションを中断してでもファーストビューを占有**して注意を促す
- **使用例**: サーバー停止予告、緊急メンテナンス通知

#### ノティフィケーションバナー
- **用途**: ウェブサイト全体/ページ/要素単位の**重要度の高い情報**を、ユーザー操作に関わらずサイト側から提示
- **使用例**: 「アップロード完了」「エラー発生」の通知
- **種類**: success / error / info / warning

#### プログレスインジケーター
- **用途**: ユーザーのアクションに対する**処理進行中の通知**
- **使用例**: 動画生成中、アップロード進捗、SUNO生成の進行

#### モーダルダイアログ
- **用途**: **重要な情報を知らせ、決定や選択を求める**。複数タスクを含む場合もある
- **使用例**: 削除確認、公開前の最終確認、APIキー入力
- **エレベーション**: レベル4 + オーバーレイシェード

---

### 📂 レイアウト・コンテナ系コンポーネント

#### アコーディオン
- **用途**: 同種のセクションが連続するとき、**折りたたんで一覧性を向上**
- **使用例**: 設定カテゴリのグループ化、FAQ

#### ディスクロージャー
- **用途**: メイン情報に対する**追加情報を折りたたんで提示**
- **使用例**: 動画情報の「詳細を表示」トグル

#### ディバイダー
- **用途**: 異なるセクション、コンポーネント、コンテンツグループ間の**視覚的な区切り**
- **使用例**: カード内のセクション分離、リストの項目区切り

#### ドロワー
- **用途**: ブラウザ画面の**四辺（上下左右端）から展開**するコンテナ。モバイルメニューなどを格納
- **使用例**: モバイルでのメインメニュー展開、サイドパネルの設定

#### 検索ボックス
- **用途**: **サイト内検索**で使用。詳細検索パネルやショートカットパーツを備える
- **使用例**: 動画タイトル検索、ログ検索

├─ Content ─────────────────────────────────────┤
│                                                │
│  ┌─ MainPanel ────────────────────────────┐   │
│  │                                         │   │
│  │  (タブごとのコンテンツ)                   │   │
│  │                                         │   │
│  └─────────────────────────────────────────┘   │

### グリッドシステム
- **12カラムグリッド**を基本（DADS準拠）
- ガター: 本文文字サイズの2倍（約 28px〜32px）
- マージン: ページ両端に確保。画面幅が狭くなっても維持

### ブレークポイント
| デバイス | ビューポート範囲 |
|---|---|
| モバイル・タブレット | 768px 未満 |
| デスクトップ | 768px 以上 |

### レスポンシブ対応
- **デスクトップ**: 2カラム（設定 + ログ）
- **タブレット**: 1カラム（設定 → ログの順）
- **スマートフォン**: 1カラム + タブをハンバーガーメニュー化（ドロワー展開）

### リキッドレイアウト
- カラムまたはガターのどちらかを可変幅とし、もう一方を固定幅にして**リキッドレイアウト**を実現
- 横スクロールが発生しても**スクロールバーを隠さない**

## インタラクション

### ボタン状態
- Default → Hover（明度+10%） → Active（明度-5%） → Disabled（opacity: 0.4）
- **Focus**: `--border-focus` の外枠線 + 背景色変化で明示（キーボード操作者のため必須）

### フォーム入力
- Empty → Focus（border: accent-primary） → Filled → Error（border: accent-danger + メッセージ）
- エラー時は `aria-invalid="true"` と関連付けエラーメッセージ（`aria-describedby`）

### ローディング
- ボタン: テキストを「処理中...」に変更 + スピナー表示
- 全体: セミトランスペアレントオーバーレイ + 中央スピナー
- プログレスインジケーターで進捗可視化（処理時間が長い場合）

### Toast通知（ノティフィケーションバナー）
- 右下に表示、3秒後に自動消去
- 種類: success(緑) / error(赤) / info(青) / warning(黄)
- 重要な通知はユーザーアクションがない限り消さない

### モーダルダイアログ
- 背景にオーバーレイシェード（`--overlay-shade`）を敷き、アンダーレイの操作を遮断
- 最前面でもボーダーを必ず設定（強制カラーモード対策）
- Escape キーで閉じる、フォーカストラップ実装

### リンクのステート
| ステート | 表現 |
|---|---|
| Default | `--link-default` + 下線 |
| Hover | `--link-hover` + 若干太い下線 |
| Active | `--link-active`（オレンジ） |
| Focus | 黒外枠線 + 黄色背景 |
| Visited | `--link-visited`（紫マゼンタ） |

## アクセシビリティ要件

### コントラスト要件（WCAG 2.2 / JIS X 8341-3:2016 準拠）

| 対象 | 最小コントラスト比 |
|---|---|
| テキスト vs 背景 | **4.5:1** 以上 |
| 非テキスト要素（アイコン、ボーダー、UIコンポーネント） | **3:1** 以上 |
| フォーカスインジケーター | 隣接色に対し **3:1** 以上 |
| アイコン（テキストシーケンス一部の可能性あり） | 背景色に対し **4.5:1** 以上 |

### ターゲットサイズ

- **24×24 CSS px 以上**（インライン、隣接ターゲットと重複なし）
- **44×44 CSS px 以上**（アイコン単体ボタン、アクション/リンクを伴うもの）

### 色覚多様性への配慮

- **色だけで情報を区別しない**（WCAG 1.4.1 レベルA）
- ステータス表示は色 + アイコン + テキストの**3つの冗長化**
- リンクは色 + 下線（テキスト装飾）で表現

### キーボード操作

- すべての操作はキーボードのみで完結可能
- **フォーカス順序とDOM順序を一致**させる
- モーダル表示中はフォーカストラップを実装

### スクリーンリーダー対応

- アイコン単独での情報提供を避け、必ず**ラベル**と組み合わせる
- ラベル併用時のアイコンには `alt=""` を指定（`aria-label` は不要）
- リンクテキストは「ここ」「こちら」ではなく具体的な内容

### リフロー・可変性対応

- **リキッドレイアウト**でテキスト200%拡大時も読める
- 文字画像ではなくテキストで情報を提供（ロゴを除く）
- `line-height` は **フォントサイズの1.5倍以上**を本文で確保

### 強制カラーモード対応

- 影（`box-shadow`）は強制カラーモードで無視されるため、**ボーダーで境界を補完**
- オーバーレイシェード上の要素には必ずボーダー（透明でも可）

### 空白・両端揃えの扱い

- `text-align: justify` は使用しない（認知障害への負担）
- 全角スペースや `&nbsp;` で文字位置を揃えない（スクリーンリーダーが単語として読めなくなる）

## 日本語UI方針

### ラベル・ボタンテキスト
- 全てのUIテキストは日本語
- 技術用語（API Key, Provider等）はそのまま英語表記可
- ボタンは動詞で統一:「生成開始」「停止」「保存」「送信」「アップロード」

### 数値表示
- 日時: `2026/04/13 20:30` 形式
- 時間: `3:00:00` （H:MM:SS）
- ファイルサイズ: `1.2 GB`

### ステータス表示
- 実行中: 緑ドット + 「実行中」
- 待機中: グレードット + 「待機中」
- エラー: 赤ドット + 「エラー」
- 完了: チェックマーク + 「完了」

### リンクテキストの書き方
- ❌ 「詳細はこちら」「もっと詳しく」
- ✅ 「動画投稿ガイドラインを読む」「Channel Bible 全文を開く」

### 数値のコントラストと強調
- ヒーロー数値（再生数、曲数等）は `--text-2xl` + `--font-bold`
- 単位は `--text-sm` で補助的に
